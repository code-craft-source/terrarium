Varieties and Subclasses


This is the second project on the way to building your own virtual Terrarium, full of digital creatures and plants. The first project can be found here. If you haven't completed that project yet, please begin there

We will use this second stage to learn about Subclasses. A subclass is a extension of an already existing class, that can be used to modify behaviour of the original. Kia, Honda, Mercedes may be subclasses of a Car class for example.

So whats so great about classes?
  • First, they modify the superclass's properties, subclasses could have different rules for defining color or width.
  • Second, theu can overwrite the superclasses ethods. Like: mySuperclassBall.move() may behave differently than mySubclassclassBall.move()

    Check out this example of a AlleyCat subclass.

    In this exercise you are going to create subclasses called EvergreenTree, MossyTree, and one your own design. Each will have unique traits and you'll use these subclasses to add variety and life to your forest, much like a real one!

    • Directions
    1. Let's first make some updates to our code to make subclassing easier. Create two new functions: showTrunk() (which shows the tree's trunk), and showLeaves() (which shows the tree's leaves). Call these new functions appropiately in your show() function ensure your forest functions the same way, and show your instructor.

    2. Now we can create our first subclass of our Tree. Each tree subclass should represent a different type of tree. Let's begin by creating a new EvergreenTree subclass. We'll modify our showLeaves() function to use p5's triangle() to create their unique appearance. Your EvergreenTree class should look something like this:

      // extends is a keyword that defines the superclass (which is Tree in this case) class EvergreenTree extends Tree { // This will overwrite Tree's showLeaves() showLeaves() { // Write new draw code to create an evergreen's unique appearance } }

    3. As you create trees in setup(), randomly select create either a Tree or an EvergreenTree. Confirm that your forest has a variety of both trees.

    4. Now let's add another subclass of Tree, MossyTree. This tree will have a moss covered trunk, giving it a green tint. This time we will overwrite Tree's showTrunk() function to create a green trunk. Your MossyTree class should look something like this:

      class MossyTree extends Tree { // This will overwrite Tree's showTrunk() showTrunk() { // Write new draw code to create a green, mossy trunk } }


    5. In your setup() function, add MossyTree to the list of trees that can be randomly selected.

    6. Come up with a new variety of tree and create your own subclass. Be creative, but think of what properties you can change to generate a new tree. You could add color, leaves, change the the size, anything! Just be sure to allow that tree to be selected in your setup() functions.

    7. Your forest should now be full of 4 different trees that each have a unique look! Next we'll observe the passage of time on your trees.
